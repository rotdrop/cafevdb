$id

ProjectEvent.php : 76
private int $id

While it would be tempting to just use the calendar URI and event UID and perhaps the recurrence id as composite key it turns out that this is complicated for repeating events: calendar apps may choose to split existing series into two when changing "this event and future" events and at that point one needs to match the recurrence ids in order to "find" the correct new old event. The event will then be part of a new event series with a new UID.
